# Legacy: roulette-advisor-ai docs (Docusaurus)

**Reference-only.** Not part of CAIA's active docs build.

## Origin

This directory is a verbatim preservation of the Docusaurus documentation site that
lived in the now-archived `prakashgbid/roulette-advisor-ai` repository at
`packages/docs/`. Original repo HEAD at archive time was
`5db0441` (2026-04-24); the GitHub `archived: true` flag was flipped on
2026-04-28.

## Why this is preserved

Per the standing capability-loss policy — _"any other repos that we archived,
we need to ensure that we do not end up losing any productive work or
functionality with it"_ (Prakash, 2026-04-28) — the docs in this site capture
non-trivial design and architecture context (architecture overview, component
guide, API reference, project structure, code style, setup/getting-started).
The audit at `reports/archived-repos-no-loss-audit-2026-04-28.md` (REM-003)
called for either preserving these docs or formally retiring them. We chose
preservation, in line with the precedents already in this `docs/` tree:

- [`docs/legacy-framework/`](../legacy-framework/) — preserved from
  `prakashgbid/framework`
- [`docs/legacy-pokerzeno-framework/`](../legacy-pokerzeno-framework/) —
  preserved from `prakashgbid/pokerzeno-framework`

## Scope of this preservation

This is **only** the documentation-site source tree. It is _not_ a port of the
roulette-advisor-ai application code. The backend tier (Mongoose models,
Express services, JWT middleware, GCP K8s manifests, etc.) is tracked
separately under REM-001 and is awaiting a product-direction call from
Prakash before any port is initiated.

### What was copied

The full `packages/docs/` tree from the archive, **minus**:

- `.docusaurus/` — autogenerated build cache (regenerable; not source)
- `node_modules/` — dependencies (regenerable via `pnpm install`)
- `package-lock.json` — npm lockfile, redundant in a pnpm workspace and
  excluded by CAIA's root `.gitignore`

That leaves 20 source files: the markdown content under `docs/`, the
Docusaurus config (`docusaurus.config.ts`, `sidebars.ts`, `tsconfig.json`),
React custom pages and CSS under `src/`, and static assets under `static/`.

### Build status

This site is **not** wired into any CAIA build pipeline. CAIA's `docs/` is
plain markdown; there is no Docusaurus build at the monorepo level. The
`package.json` here references `@docusaurus/*` packages, but because CAIA's
`pnpm-workspace.yaml` only globs `apps/*`, `packages/*`, `configs/*`,
`templates/*`, this folder is _not_ a workspace member and `pnpm install` at
the CAIA root will not attempt to install its dependencies.

If anyone ever wants to build this site standalone, they can `cd` into this
directory and run `npm install && npm run build` against a Node 18+ toolchain
— but that's an explicit ad-hoc operation, not part of CI.
